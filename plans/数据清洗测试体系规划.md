# 数据清洗测试体系规划

## 背景

`clean_anki_data.py` 是核心数据清洗程序：
- 输入：`mydata.anki2`（Anki 数据库）
- 输出：`plains/cleaned_data.csv`（word / segmentation / related_words 三列）
- 核心函数：`clean_html()`、`extract_segmentation()`、`extract_related_words()`

当前没有任何正式的测试文件和测试数据集，之前调试用的临时脚本已全部删除。

## 要创建的文件

### 1. `tests/test_data.py` — 测试数据集

硬编码的手工构造 field3 样例，覆盖各种边界情况。

| 用例 | word | 构造的 field3 | 期望 seg | 期望 related |
|------|------|---------------|----------|-------------|
| 标准点号切分 | respectively | `re.spec.tive.ly 相关词` | `re.spec.tive.ly` | — |
| +号分隔相关词 | extradition | `ex+tradition 传统` | `ex.tradition` | `tradition` |
| 汉字+ + 号 | moratorium | `mor死亡ator+ium ... san+ator.ium crem+ator.ium` | `mor.ator.ium` | `san+ator.ium`; `crem+ator.ium` |
| 纯字母相关词 | coral | `coral 珊瑚 corral carol` | — | `corral`; `carol` |
| HTML 标签 | heat | `he<br>at 热` | `he.at` | — |
| 无切分无相关词 | tyrant | 无相关内容 | — | — |
| 子串噪声 | moratorium 例 | `-ator.ium` 是词缀注释，不应被提取 | — | 不应有 `atorium` |

### 2. `tests/test_clean.py` — 测试运行脚本

从 `test_data.py` 加载测试用例，逐条调用 `clean_html()`、`extract_segmentation()`、`extract_related_words()` 验证结果。

运行方式：
```bash
python tests/test_clean.py
```

输出格式：
```
✅ respectively    seg=re.spec.tive.ly
✅ extradition    seg=ex.tradition  related=tradition
✅ moratorium     seg=mor.ator.ium  related=san+ator.ium; crem+ator.ium
✅ coral          related=corral; carol
✅ heat           seg=he.at
✅ tyrant         seg=  related=
...
通过: 6/6
```

## 目录结构

```
查词拆词/
├── clean_anki_data.py     ← 主程序（被测试）
├── plains/
│   └── cleaned_data.csv   ← 输出数据
├── tests/                  ← 新建
│   ├── __init__.py
│   ├── test_data.py        ← 测试数据集
│   └── test_clean.py       ← 测试运行器
```

## 注意事项

1. 测试数据不依赖 mydata.anki2，用硬编码的字符串，确保可复现
2. 测试运行时不会覆盖实际的 `plains/cleaned_data.csv`
3. 每次修改 `clean_anki_data.py` 的核心函数后，跑 `python tests/test_clean.py` 确认
