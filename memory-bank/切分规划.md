## ⚙️ 核心架构规划

这个程序的核心逻辑可以拆分为两个双轨驱动的引擎：**结构切分引擎**和**形似词检索引擎**。

```
[用户输入单词] 
       │
       ├──> 轨道 A：结构切分 ENGINE (词根、词缀字典 + 算法) ──> 输出: in.veter.ate
       │                                                          │
       └──> 轨道 B：形似词检索 ENGINE (模糊匹配 + 词频过滤)        │ (联动扩展)
                               │                                  ▼
                     [召回形似候选词] ───────────────────────> [结合已切分词根]
                               │                                  │
                               ▼                                  ▼
                     按 COCA 词频降序排列                   输出: veter.an
                               │                                  bene.vol.ent
                               ▼
                        展示最终对比列表 (th.ong / pr.ong)

```

## 🛠️ 详细实施方案

### 1. 数据准备（你的核心资产）

你已有的 1000+ 条数据是极好的**种子数据集（Seed Dataset）**。

* **用途 1（规则提炼）：** 提取出你已经切分好的常见前缀（`in-`, `bene-`）、后缀（`-ent`, `-ate`）和词根（`vol`, `veter`），形成基础特征库。
* **用途 2（算法校验）：** 留出 100-200 条作为测试集，用来检验后面算法切得准不准。

#### 1.1 数据清洗（重点）
由于 Anki 数据质量参差不齐，需要进行以下清洗步骤：
- **去重**：去除重复条目
- **标准化**：统一单词格式（小写、去除空格、特殊字符）
- **格式校验**：验证切分格式的正确性（如 `prefix.stem.suffix` 格式是否规范）
- **异常处理**：处理空字段、无效切分、乱码等异常数据
- **质量评估**：建立数据质量指标（清洗率、格式正确率、有效条目数）

### 2. 轨道 A：结构切分引擎（Segmentation）

处理有明确词根词缀的单词（如 `male.vol.ent`）。

* **实现方法：** **基于字典的规则匹配 + 正则表达式**。
* **逻辑：**
1. 构建一个高频词缀词根表。
2. 从两头向中间剥离：先匹配最长的前缀，再匹配最长的后缀，剩下的部分交付词根库校验。
3. 如果剩余部分在词根库中，则切分成功。

### 3. 轨道 B：形似词检索引擎（Fuzzy Matching）

处理没有清晰词根或需要找形似词的单词（如 `thr.ong` $\rightarrow$ `thr.one`）。

这是这个程序的难点和亮点，建议使用以下组合拳：

* **第一步：编辑距离算法（Levenshtein Distance）**
允许增删改 1-2 个字母。例如 `snub` 和 `snob` 的编辑距离是 1。

* **第二步：骨架匹配（Sorensen-Dice / n-gram）**
对于 `thr.ong` 和 `pr.ong`，它们共享了末尾的 `ong` 字母簇。你可以通过提取单词的辅音丛（Consonant Clusters）或固定尾缀（如 `-ong`, `-ub`）来进行桶分类（Bucketing），快速捞出同尾/同首词。

* **第三步：基于切分结果的反向检索**
如果轨道 A 识别出 `inveterate` 含有词根 `veter`，直接去词典库库里检索所有包含 `veter` 子字符串的单词（如 `veteran`），直接并入形似词列表。


### 4. 排序过滤（Ranking）

捞出来的形似词可能有很多，必须过滤和排序，否则信息过载。

* **词频过滤：** 接入开源的词频表（如 **COCA 6万词频表** 或 **Google Books Ngram**）。只保留词频在前 **40,000** 以内的常用词，生僻词直接扔掉。
* **排序机制：** 综合"形似度分值（70%）" + "词频分值（30%）"进行最终降序排列，确保用户先看到常用词。

## 🚀 技术栈与开发建议

如果你打算自己开发原型，推荐使用 **Python**，生态最成熟：

* **文本处理与模糊匹配：** `textdistance` 或 `Levenshtein` 库（计算编辑距离）；`NLTK` 库（进行词性标注和基础文本处理）。
* **词频数据：** 寻找开源的 `COCA_frequency_list.csv`，导入作为过滤字典。
* **数据存储：** 现阶段用 `SQLite` 或直接读写 `JSON/CSV` 文件即可，方便存储你那 1000+ 条种子数据。
